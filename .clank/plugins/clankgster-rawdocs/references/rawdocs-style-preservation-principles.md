# Rawdocs style preservation principles

This reference defines style-preservation expectations for docsraw analysis and sync workflows.

## Principle: source voice is primary

The author's source language in `rawdocs/` is intentional by default.

Sync workflows should preserve that intent and voice. Creativity for rephrasing should remain near zero unless clarity or correctness requires a small adjustment.

## Preserve these signals

- heading and section style,
- sentence cadence and tone,
- habitual terminology choices,
- quote-style conventions in examples where observable,
- list and outline patterns.

## Editing posture

- Prefer porting source phrasing as-is.
- Apply spelling and grammar edits sparingly and only with clear benefit.
- Do not overwrite distinct voice with generic polished prose.

## Structural transformation vs textual transformation

This system is primarily structural.

- **Allowed:** reorganize file boundaries, grouping, and placement.
- **Constrained:** rewrite source wording.

## Conflict handling

When source style and existing structure conflict:

1. preserve source meaning and style,
2. preserve continuity where possible,
3. evolve structure when needed for long-term maintainability.

Document rationale explicitly in the refined plan.
