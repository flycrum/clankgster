# Common steps: content quality audit

1. Resolve target and confirm it exists.
2. Read all relevant markdown files for the target.
3. Evaluate per file:
   - clarity and concision
   - framing (prefer positive alternatives)
   - emphasis calibration
   - example quality
   - inferable/redundant content
4. Evaluate `SKILL.md` description quality when present.
5. Produce severity-ranked findings and overall grade.
